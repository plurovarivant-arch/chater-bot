# Инструкция по запуску бота на Railway (бесплатно)

## Шаг 1 — Создай бота в Telegram

1. Открой Telegram, найди @BotFather
2. Напиши /newbot
3. Придумай имя боту (напр. ChaterTest)
4. Придумай username (напр. mychatertest_bot)
5. BotFather пришлёт тебе **токен** — скопируй его, он понадобится

## Шаг 2 — Узнай свой chat_id

1. Найди бота @userinfobot в Telegram
2. Напиши ему /start
3. Он пришлёт твой **Id** — это и есть твой ADMIN_CHAT_ID

## Шаг 3 — Залей код на GitHub

1. Зайди на github.com, создай аккаунт если нет
2. Нажми "New repository", назови например chater-bot
3. Загрузи три файла: bot.py, requirements.txt, Procfile

Содержимое Procfile (создай файл с таким именем, без расширения):
```
worker: python bot.py
```

## Шаг 4 — Задеплой на Railway

1. Зайди на railway.app
2. Войди через GitHub
3. Нажми "New Project" → "Deploy from GitHub repo"
4. Выбери свой репозиторий chater-bot
5. После деплоя перейди в Variables (настройки переменных)
6. Добавь две переменные:
   - BOT_TOKEN = (вставь токен от BotFather)
   - ADMIN_CHAT_ID = (вставь свой chat_id)
7. Railway автоматически перезапустит бота

## Шаг 5 — Готово!

Бот запущен. Теперь:
- Ты отправляешь кандидату ссылку на бота (напр. t.me/mychatertest_bot)
- Кандидат нажимает /start, вводит имя и проходит 5 заданий
- Каждый ответ сразу приходит тебе в личку в Telegram
- В конце ты получишь полный отчёт со всеми ответами

## Как отличить кандидатов?

Бот спрашивает имя у каждого кандидата в начале.
В каждом сообщении тебе будет указано: "Кандидат: Имя"

## Проблемы?

- Бот не отвечает → проверь BOT_TOKEN в Railway переменных
- Ответы не приходят → проверь ADMIN_CHAT_ID (должен быть числом, без кавычек)
- Ошибка на Railway → посмотри логи во вкладке Deployments
